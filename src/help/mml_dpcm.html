NSDL（NES Sound Driver Library）

トップ＼ 使い方＼ MMLコンパイラ＼ MML文法＼

MML reference: Delta PCM define block

定義（Define）

DPCM{
	nnumber,"filename",mode(,volume)(,next)
	note(Accidentals)octave,"filename",mode(,volume)(,next)
}

@DPCM{
	nnumber,"filename",mode(,volume)(,next)
	note(Accidentals)octave,"filename",mode(,volume)(,next)

引数（Inputs）

number

定義する⊿PCMの番号を0～254で指定してください。
⊿PCMの発音の際は、ノートナンバーの値を参照し、その番号の⊿PCMを発音します。

notenote number
o1c 0
o1c+1
o1d 2
o1d+3
o1e 4
o1f 5
o1f+6
o1g 7
o1g+8
o1a 9
o1a+10
o1b 11
o2c 12
・・・ ・・・

note

⊿PCMの波形を割り当てる音階を、音階で指定します。

a …
A, La, ラ
b …
H, Si, シ
c …
C, Do, ド
d …
D, Re, レ
e …
E, Mi, ミ
f …
F, Fa, ファ
g …
G, Sol, ソ

Accidentals

⊿PCMの波形を割り当てる音階に対する臨時記号です。省略可能です。

+ …
♯ Sharp
# …
♯ Sharp
- …
♭ Flat
= …
♮ Natural

octave

⊿PCMの波形を割り当てるオクターブです。1～8で指定してください。

filename

割り当てる⊿PCMの波形ファイル（*.dmc）を指定してください。

mode

0 …
1 shot
1 …
loop
2 …
IRQ （発音終了時にnextで指定したノート番号の⊿PCMを発音します。）

volume

⊿PCMの初期ボリュームです。
0～127の範囲で指定した場合は、波形再生開始時に$4011番地に書く初期ボリュームとなります。
-1を指定した場合は、波形再生開始時に$4011番地に何も書きません。
省略した場合は、0となります。

next

発音終了時に次にならすノート番号です。
バンクに対応したbin（nsd_all.bin, nsd_all2.bin）でのみ対応しています。

内容（Contents）

指定の各音階に、⊿PCMを割り当てます。

注意（Notes）

⊿PCMを割り当てる場合は、 #OffsetPCMコマンドにて⊿PCMの波形データの配置アドレスを設定してください。
⊿PCMは、設定したアドレスから、0xFFFFのメモリ空間に配置されます。
若しくは、#Bankコマンドを記述しバンクに対応したbin（nsd_all.bin, nsd_all2.bin）を使用して下さい。

また、多くのNSFプレイヤーは、IRQに対応しておりません。
以下、2013年 2月14時点での対応状況です。

IRQ対応プレイヤ

  • VirtuaNSF
  • VirtuaNES
  • Nintendulator

IRQ非対応プレイヤ

  • TNS-HFC3　…（割り込みベクタを全て書き換えてしまうため）
  • TNS-HFC4　…（割り込みベクタを全て書き換えてしまうため）　
  • NSFPlay　…（IRQそのものが実装されていません。）
  • NEZPlug　…（IRQが無効になっています。）
  • hoot　…（IRQが無効になっています。）

※実機での再生環境TNS-HFC3でもIRQに対応しておりません。
これは、6502の割り込みベクタをIRQ含め全て書き換えてしまうためです。
IRQの割り込みベクトル先は、RTI命令のみになっているとのことです。
また、TNS-HFC4でも非対応とのことですが、検討課題として頂きました。
